# v1.1 Integration Readiness

## Scope

v1.1 readiness covers the shared public command surface across Claude Code, Codex, and OpenCode. The release candidate is evaluated through `search`, `show`, `export`, and `doctor`, with deterministic fixture-mode evidence available through `AGENTSCOPE_FIXTURES_MODE=1`.

This readiness note validates integration coverage. It does not add a new command, persistent index, dashboard, queryless browse mode, or new runtime-adapter capability.

## Command Matrix

| Command | Readiness expectation |
|---------|----------------------|
| `agentscope search proxy --json` | Returns bounded JSON search results across available runtimes, preserves partial results when one runtime is unavailable, and emits runtime-scoped warnings. |
| `agentscope show child-019d --json` | Resolves exact or safe partial session IDs, reports ambiguity with structured candidates, and materializes a temporary bundle without printing full transcript bodies. |
| `agentscope export child-019d --out <directory>` | Writes an explicit local bundle with `manifest.json`, native payload files, provenance, and safe warning metadata. |
| `agentscope doctor --json` | Reports Claude Code, Codex, and OpenCode detection, store status, confidence signals, sanity checks, and warnings in a stable JSON envelope. |

## Known Limitations

- Codex storage is observed behavior, not a stable public API.
- Codex parent/root linkage remains conservative when durable metadata is absent.
- OpenCode `tool-output/` completeness remains a known limitation.
- Raw/native payloads stay in explicit local bundles and are not printed as full transcript bodies to stdout.

## Release Decision

v1.1 is ready to approve only after `npm test` and `npm run build` both pass. Any blocking regression found during those commands should be fixed before approval; non-blocking runtime caveats should remain documented in the known limitations above and in runtime-specific docs.
